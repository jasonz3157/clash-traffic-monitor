# v2.1.2

## Highlights

- 自动切换支持递归切换已启用的嵌套策略组
  - 当大流量命中某个已启用策略组后，会先切换这个命中组
  - 如果它的目标本身也是一个已启用的自动切换策略组，则会继续递归切换下一层
  - 递归会一直持续到目标不是“已启用的自动切换策略组”为止
- 遇到未启用的目标组时，停在该组上并视为成功
  - 如果上一级目标是一个策略组，但这个组没有在自动切换配置里启用
  - 系统会停在这个组上，不再继续往下切
  - 这种情况不会算作错误
- 增加递归链路保护与测试覆盖
  - 新增循环检测，防止 `A -> B -> A` 这类递归配置导致死循环
  - 新增测试覆盖递归切换、停在未启用组、循环保护三类场景

## Behavior

- 触发条件仍然是“某个域名命中的策略组，在 1 分钟窗口内累计流量超过阈值”。
- 自动切换从这次流量实际命中的策略组开始执行。
- 如果 `🌍 国外媒体 -> ⏬ 大流量套餐`，且 `⏬ 大流量套餐 -> 🇸🇬 Singapore`，并且这两个组都在自动切换里启用，则触发后会依次切换这两层。
- 如果 `🌍 国外媒体 -> ⏬ 大流量套餐`，但 `⏬ 大流量套餐` 没有在自动切换里启用，则只切换 `🌍 国外媒体` 到 `⏬ 大流量套餐`，然后停止，并将本次切换视为成功。
- 如果检测到递归链路存在循环引用，会记录错误并停止后续递归。
